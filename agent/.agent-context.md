## ðŸ“œ Conventions
- **Session State**: Use the `AgentSession` interface. History should be a list of role/content pairs.
- **Agent Definitions**: All agents must be defined in the `PREDEFINED_AGENTS` constant in `index.ts`.
- **Headless Execution**: Use `runAgentTask` for spawned subagents (no Discord context).
 - **Authorization checks**: High-risk agents are owner-only (currently `OWNER_ID` or `DEFAULT_MENTION_USER_ID`).

## ðŸ§  Collective Knowledge
- **Risk Levels**: High-risk agents (e.g., `cursor-fast`) often have `force: true` and should be used with caution.
- **Current limitation**: `runAgentTask()` does **not** support the default `'claude'` client headlessly yet (Cursor/Antigravity only).
- **Context files security**: any feature that reads `context_files` must be path-safe (see repo security issues like #1/#39).
- **Prompt injection**: multiple open issues exist around prompt/history handling (#41/#50).
- **Context Injection**: Use `include_system_info` and `context_files` sparingly; avoid including secrets or large diffs.
- **Gemini 2.0 Flash**: The default model for the Manager due to its speed and logic prowess.
